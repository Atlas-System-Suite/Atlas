# ADR-003: Framework-Agnostic Core

Status: Accepted

Date: 2026-06-28

---

## Context

Atlas needs to support multiple application surfaces: CLI, web, desktop, and mobile.

The question is whether Atlas Core should embed a web framework (e.g., FastAPI) or remain framework-agnostic.

---

## Decision

Atlas Core is a library. It does not embed or depend on any web framework.

Applications built on Atlas choose their own presentation layer:

* CLI applications use Click, Typer, or argparse.
* Web applications use FastAPI, Django, Litestar, or any other framework.
* Desktop applications use Tauri, Electron, or native frameworks.
* Mobile applications use appropriate mobile frameworks.

Atlas Core provides:

* Runtime (boot, lifecycle, shutdown)
* Event Bus
* Registry and Capability resolution
* Configuration
* Module and Provider management

Atlas Core does NOT provide:

* HTTP routing
* Request/response handling
* Template rendering
* Authentication middleware
* Session management

These are the responsibility of the Application layer.

---

## Rationale

* **Framework lock-in is the #1 risk** for a long-lived framework. FastAPI could be replaced by a better alternative in 2 years.
* **Multiple surfaces require multiple frameworks.** A CLI app and a web app cannot share FastAPI.
* **Core simplicity.** Embedding a web framework adds dependencies, opinions, and complexity that most Atlas applications may not need.
* **The architecture already supports this.** Applications assemble Modules and Providers — they can also assemble a web framework.

---

## Alternatives Considered

### Embed FastAPI

Would provide a ready-made web surface but would couple Core to HTTP concerns. Rejected because it violates the "Applications decide their surface" principle.

### Provide an Optional Web Package

A `atlas-web` package could provide FastAPI integration. This is a valid future direction but not needed for Core.

---

## Consequences

* Atlas Core has no web-related dependencies.
* The SDK should provide templates for different application surfaces.
* Example applications should demonstrate how to integrate Atlas with FastAPI, Click, etc.
* No HTTP-related code exists in `src/atlas/core/`.
