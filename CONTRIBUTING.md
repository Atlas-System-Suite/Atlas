# Contributing Guidelines

Thank you for your interest in contributing to Atlas! We welcome bug reports, feature suggestions, documentation improvements, and code contributions.

## How to Contribute

### Reporting Bugs
- Check existing issues before creating a new report.
- Include detailed steps to reproduce the issue along with expected and actual behaviors.
- Specify your system environment and Atlas version.

### Suggesting Features
- Open an issue with a clear title and detailed description of the proposed functionality and motivation.

### Development Workflow
1. Fork the repository and create a new branch for your feature or bug fix:
   ```bash
   git checkout -b feature/your-feature-name
   ```
2. Write clean, well-tested code adhering to standard formatting and conventions.
3. Commit your changes with clear, descriptive commit messages.
4. Push your branch and open a Pull Request against the main branch.

## Code Style & Standards
- Keep code clean, readable, and consistent.
- Ensure proper error handling and documentation for new APIs or syntax additions.

## Code of Conduct
Please note that this project is governed by our [Code of Conduct](CODE_OF_CONDUCT.md). By participating, you agree to uphold this code.
