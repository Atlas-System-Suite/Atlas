# ADR-001: Python with Hybrid Async Runtime

Status: Accepted

Date: 2026-06-28

---

## Context

Atlas requires a programming language and runtime model for its Core framework.

The runtime model decision affects every interface, Protocol, and Provider in the system.

---

## Decision

### Language: Python 3.13+

Atlas is implemented in Python.

### Runtime Model: Hybrid Async

* The Atlas Runtime uses `async/await` internally for all I/O-bound operations.
* All Core Protocol methods use `async def`.
* Sync convenience wrappers are provided for simple use cases.
* `asyncio.run()` is used only at entry-point boundaries.

---

## Rationale

### Why Python

* Rich ecosystem for AI integration (OpenAI, Anthropic, Google SDKs are Python-first).
* Strong typing via Protocols, type hints, and tools like BasedPyright.
* `asyncio` provides native async support without external frameworks.
* Large community and hiring pool.
* Mature package ecosystem for all target domains (data, web, automation).

### Why Hybrid Async

* Atlas Core is I/O-heavy: storage, AI, notifications, and external APIs all benefit from async.
* Pure async would force all consumers to understand `asyncio`, increasing adoption friction.
* Sync wrappers allow simple scripts and CLIs to use Atlas without async boilerplate.
* The hybrid model follows patterns used by `httpx`, `playwright`, and `prisma-client-py`.

---

## Alternatives Considered

### Pure Synchronous

Simpler but would create performance bottlenecks for AI providers and network I/O. Would require threads for concurrency, adding complexity elsewhere.

### Pure Async

Better performance but higher cognitive overhead. Would make the SDK harder for beginners.

### TypeScript / Node.js

Strong async ecosystem but weaker AI/ML library support. Would limit local model integration.

### Rust

Excellent performance but slower development velocity. Too early for Atlas to trade development speed for runtime speed.

---

## Consequences

* All Protocols must define `async def` methods.
* Testing requires `pytest-asyncio`.
* Contributors must understand basic `asyncio` patterns.
* Sync wrappers must be maintained alongside async implementations.
* CodingStandards must include async/await conventions.
